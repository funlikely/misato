# みさと / Misato

A Japanese-speaking dating-sim style desktop chatbot. Misato (みさと) is a confident, playful, openly flirtatious adult woman in her late 20s. She **always speaks natural casual Japanese** — that's her real voice. Every line is shown with an English subtitle below for you to read along, which you can hide if you don't need it.

You can talk to her by typing **or by speaking** — a mic button in the composer uses the webview's built-in speech recognition (Japanese or English; switchable).

Runs locally with [Ollama](https://ollama.com) — no cloud, no API keys, no logs leaving your machine.

> Tone: suggestive and flirty, fade-to-black at the explicit line. Adults only.

## Modes

- **フリートーク / Freeform** — just chat.
- **好感度モード / Affection** — chat plus a 0–100 affection meter she reacts to.
- **ビジュアルノベル / VN** — scene-based; she'll occasionally offer multiple-choice branches.

You pick a mode on launch and can switch back anytime.

## For users

### 1. Install prerequisites (Windows)

```powershell
# Rust toolchain (for Tauri)
winget install --id Rustlang.Rustup -e

# Microsoft C++ build tools (Tauri needs them on Windows)
winget install --id Microsoft.VisualStudio.2022.BuildTools -e --override "--add Microsoft.VisualStudio.Workload.VCTools --includeRecommended --quiet"

# Ollama (local LLM runtime)
winget install --id Ollama.Ollama -e
```

Open a fresh shell so PATH picks up the new tools, then pull a Japanese-capable model:

```powershell
ollama pull qwen2.5:7b
```

`qwen2.5:7b` is a good default — handles Japanese well, runs on modest hardware. If you have a beefy GPU, `qwen2.5:14b` is noticeably better; edit `DEFAULT_CONFIG.model` in `src/lib/ollama.ts`.

### 2. Install JS dependencies

```powershell
npm install
```

### 3. Run

```powershell
npm run tauri:dev
```

The first Rust build takes 5–10 minutes (compiling Tauri + crates). Subsequent runs are fast and the frontend hot-reloads.

Make sure Ollama is running in the background — it usually auto-starts after install. Test with `ollama list`.

### Build a standalone .exe / installer

```powershell
npm run tauri:build
```

Output lands in `src-tauri/target/release/bundle/`.

## For developers

### Stack

- **Tauri 2** — desktop shell (Rust)
- **React 18 + TypeScript + Vite** — frontend
- **Ollama HTTP API** — LLM backend, streamed via `fetch` from the webview directly (no Rust-side HTTP code)
- No database, no global state library — just `useState` + `localStorage`

### The LLM contract

Misato's system prompt requires every reply to follow this exact shape:

```
[mood:flirty][affection:+2]
[ja] そういうの好きだなぁ、私。
[en] Mmh… I kinda like that side of you.
[choice] もう少し聞かせて
[choice] 「やめてよ恥ずかしい」
```

Tags consumed by the frontend:

| Tag | Where | Used by |
|---|---|---|
| `[mood:X]` | always, leading | `Portrait` swaps expression |
| `[affection:±N]` | affection / vn modes | adjusts meter, -5 to +5 |
| `[ja] ...` | always | rendered as the main spoken line |
| `[en] ...` | always | rendered as English subtitle (toggle-able) |
| `[choice] ...` | vn mode, optional | rendered as branch buttons |

Parsing lives in `src/lib/moods.ts` (`parseMood`, `parseAffectionDelta`, `parseBilingual`) and `src/components/Chat.tsx` (`parseChoices`, `parseAssistant`).

Allowed moods live in `src/lib/moods.ts` (`MOODS`). Add to that array + `MOOD_EMOJI` + `MOOD_COLOR` to introduce a new expression.

### Persona

The character voice is in `src/persona/misato.ts`. `systemPromptFor(mode, state)` assembles:

1. `CORE` — voice, language rules, output format, tone guardrails
2. `AFFECTION_RULES` — appended in `affection` and `vn` modes
3. `VN_RULES` — appended in `vn` mode only
4. `[state]` block — current affection score and (in VN) scene info

If you want a different archetype (tsundere, onee-san, quiet/dere), edit `CORE` rather than forking — most of the surrounding plumbing is archetype-agnostic.

### Switching models

Edit `DEFAULT_CONFIG` in `src/lib/ollama.ts`:

```ts
export const DEFAULT_CONFIG: OllamaConfig = {
  host: "http://localhost:11434",
  model: "qwen2.5:14b",       // ← change this
  temperature: 0.85,
};
```

Any Ollama model that handles Japanese works. Candidates worth trying:

- `qwen2.5:7b` / `qwen2.5:14b` — Alibaba, strong JP, default
- `gemma2:9b` — Google, decent JP
- `llama3.1:8b` — okay JP, very permissive personality

Run `ollama list` to see what's pulled locally.

### Sprites

`Portrait.tsx` currently renders a colored emoji placeholder per mood. To use real art:

1. Drop PNGs into `public/portraits/<mood>.png` (e.g. `flirty.png`, `embarrassed.png`).
2. In `Portrait.tsx`, replace the `<div className="portrait-emoji">` with `<img src={`/portraits/${mood}.png`}>`.
3. Optional: add a default/fallback for missing moods.

### VN scenes

Currently the VN `[state]` line is hardcoded to a single scene (`喫茶店 (afternoon)` in `Chat.tsx → buildStateLine`). A real scene system would:

- Track current scene/location/time in component state
- Let Misato change scenes via a `[scene:X]` tag or explicit choices
- Render a background image per scene

The hook point is `buildStateLine` in `src/components/Chat.tsx`.

### Reset

Click **Reset** in the sidebar, or clear `misato.session` from localStorage in devtools. Persistence is single-session — re-picking a mode loads the matching saved session.

### Regenerating icons

If you want to change the app icon:

```powershell
# Replace icon-source.png with your own 1024×1024 PNG, then:
npx tauri icon icon-source.png
```

This regenerates everything in `src-tauri/icons/`.

## Troubleshooting

**"Ollama 500 / connection refused"** — Ollama isn't running. Start it (`ollama serve` or via the tray app) and confirm `curl http://localhost:11434/api/tags` works.

**"Model not found"** — `ollama pull <model>` first, or change `DEFAULT_CONFIG.model` to one you have.

**First `tauri:dev` is slow / hangs** — it's not hung; Rust is compiling ~400 crates. Subsequent runs are seconds, not minutes.

**Window is blank** — open devtools (right-click → Inspect) and check the console. Most blank-screen issues are TS/import errors in `src/`.

**Japanese looks like boxes** — system is missing Japanese fonts. On Windows, install "Japanese Supplemental Fonts" via Settings → Apps → Optional features.

**Mic button does nothing / errors immediately** — the webview's `webkitSpeechRecognition` likely isn't reachable from inside WebView2. Fallback options (whisper.cpp, etc.) are tracked in [`docs/todo.md`](docs/todo.md).
