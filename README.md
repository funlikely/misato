## For users

### 1. Install prerequisites (Windows)

```powershell
# Node.js LTS (frontend tooling ‚Äî needs Node 18+, tested on 22)
winget install --id OpenJS.NodeJS.LTS -e

# Rust toolchain (for Tauri)
winget install --id Rustlang.Rustup -e

# Microsoft C++ build tools (Tauri needs them on Windows)
winget install --id Microsoft.VisualStudio.2022.BuildTools -e --override "--add Microsoft.VisualStudio.Workload.VCTools --includeRecommended --quiet"

# Ollama (local LLM runtime)
winget install --id Ollama.Ollama -e
```

Open a fresh shell so PATH picks up the new tools, then pull the default model:

```powershell
ollama pull lucas2024/llama-3-elyza-jp-8b:q5_k_m
```

This is ELYZA's Japanese fine-tune of Llama 3 ‚Äî it handles natural casual Japanese well and, unlike some general models, doesn't leak Chinese characters. It runs on modest hardware. To use a different model, edit `DEFAULT_CONFIG.model` in `src/lib/ollama.ts` (see [Switching models](#switching-models)).

### 1b. Install VOICEVOX (for her voice)

Misato speaks her Japanese lines aloud via [VOICEVOX](https://voicevox.hiroshiba.jp/), a free local TTS engine. **The app works without it** (text only), but for voice you need to download and run VOICEVOX separately ‚Äî it's not a `winget` package:

1. Download the VOICEVOX installer from <https://voicevox.hiroshiba.jp/> (the CPU build is fine; a GPU build is faster).
2. Launch the VOICEVOX app. It serves a local engine on `http://localhost:50021`.
3. Keep it open while using Misato. The sidebar shows the engine status; if it reads "VOICEVOX not running," the app is closed or still starting.

### 2. Install JS dependencies

```powershell
npm install
```

### 3. Run

```powershell
npm run tauri:dev
```

The first Rust build takes 5‚Äì10 minutes (compiling Tauri + crates). Subsequent runs are fast and the frontend hot-reloads.

Make sure Ollama is running in the background ‚Äî it usually auto-starts after install. Test with `ollama list`.

### Build a standalone .exe / installer

```powershell
npm run tauri:build
```

Output lands in `src-tauri/target/release/bundle/`.

### The LLM contract

Misato's system prompt requires every reply to follow this exact shape:

```
[mood:flirty][affection:+2]
[ja] „Åù„ÅÜ„ÅÑ„ÅÜ„ÅÆÂ•Ω„Åç„ÅÝ„Å™„ÅÅ„ÄÅÁßÅ„ÄÇ

[choice] „Äå„ÇÑ„ÇÅ„Å¶„ÇàÊÅ•„Åö„Åã„Åó„ÅÑ„Äç
```

Tags consumed by the frontend:

| Tag | Where | Used by |
|---|---|---|
| `[mood:X]` | always, leading | `Portrait` swaps expression |
| `[affection:¬±N]` | affection / vn modes | adjusts meter, -5 to +5 |
| `[ja] ...` | always | rendered as the main spoken line |
| `[en] ...` | always | rendered as English subtitle (toggle-able) |
| `[choice] ...` | vn mode, optional | rendered as branch buttons |

Parsing lives in `src/lib/moods.ts` (`parseMood`, `parseAffectionDelta`, `parseBilingual`) and `src/components/Chat.tsx` (`parseChoices`, `parseAssistant`).

Allowed moods live in `src/lib/moods.ts` (`MOODS`). Add to that array + `MOOD_EMOJI` + `MOOD_COLOR` to introduce a new expression.

### Persona

The character voice is in `src/persona/misato.ts`. `systemPromptFor(mode, state)` assembles:

1. `CORE` ‚Äî voice, language rules, output format, tone guardrails
2. `AFFECTION_RULES` ‚Äî appended in `affection` and `vn` modes
3. `VN_RULES` ‚Äî appended in `vn` mode only
4. `[state]` block ‚Äî current affection score and (in VN) scene info

If you want a different archetype (tsundere, onee-san, quiet/dere), edit `CORE` rather than forking ‚Äî most of the surrounding plumbing is archetype-agnostic.

### Switching models

Edit `DEFAULT_CONFIG` in `src/lib/ollama.ts`:

```ts
export const DEFAULT_CONFIG: OllamaConfig = {
  host: "http://localhost:11434",
  model: "lucas2024/llama-3-elyza-jp-8b:q5_k_m",   // ‚Üê change this
  temperature: 0.85,
};
```

Any Ollama model that handles Japanese works. Candidates worth trying:

- `lucas2024/llama-3-elyza-jp-8b:q5_k_m` ‚Äî ELYZA's Llama-3 JP fine-tune, strong natural JP, no Chinese-script bleed, **default**
- `qwen2.5:7b` / `qwen2.5:14b` ‚Äî Alibaba, strong JP (can occasionally leak Chinese characters)
- `gemma2:9b` ‚Äî Google, decent JP
- `llama3.1:8b` ‚Äî okay JP, very permissive personality

Run `ollama list` to see what's pulled locally.

### Sprites

`Portrait.tsx` currently renders a colored emoji placeholder per mood. To use real art:

1. Drop PNGs into `public/portraits/<mood>.png` (e.g. `flirty.png`, `embarrassed.png`).
2. In `Portrait.tsx`, replace the `<div className="portrait-emoji">` with `<img src={`/portraits/${mood}.png`}>`.
3. Optional: add a default/fallback for missing moods.

Click **Reset** in the sidebar, or clear `misato.session` from localStorage in devtools. Persistence is single-session ‚Äî re-picking a mode loads the matching saved session.

## Troubleshooting

**"Ollama 500 / connection refused"** ‚Äî Ollama isn't running. Start it (`ollama serve` or via the tray app) and confirm `curl http://localhost:11434/api/tags` works.

**"Model not found"** ‚Äî `ollama pull <model>` first, or change `DEFAULT_CONFIG.model` to one you have.

**First `tauri:dev` is slow / hangs** ‚Äî it's not hung; Rust is compiling ~400 crates. Subsequent runs are seconds, not minutes.

**Window is blank** ‚Äî open devtools (right-click ‚Üí Inspect) and check the console. Most blank-screen issues are TS/import errors in `src/`.

**Japanese looks like boxes** ‚Äî system is missing Japanese fonts. On Windows, install "Japanese Supplemental Fonts" via Settings ‚Üí Apps ‚Üí Optional features.

**No voice / sidebar says "VOICEVOX not running"** ‚Äî the VOICEVOX app isn't open. Launch it and confirm `curl http://localhost:50021/version` responds. The app falls back to text-only until the engine is reachable. See [step 1b](#1b-install-voicevox-for-her-voice).

**Mic button does nothing / errors immediately** ‚Äî the webview's `webkitSpeechRecognition` likely isn't reachable from inside WebView2. Fallback options (whisper.cpp, etc.) are tracked in [`docs/todo.md`](docs/todo.md).
